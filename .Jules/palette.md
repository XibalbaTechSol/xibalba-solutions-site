## 2025-10-26 - Form Submission UX and Mobile Accessibility
**Learning:** Standard HTML forms (like those using Formspree) often lack visual feedback during submission, leading to user uncertainty and potential double-submissions. Adding a simple global event listener for `submit` events on forms with an `action` attribute is a highly effective, low-impact way to improve this across a static site. Also, managing `aria-expanded` on mobile menus is a critical but often overlooked accessibility feature that is easy to implement in vanilla JS.
**Action:** In future static sites, implementing a global form submission handler that disables the submit button and changes text to "Sending..." should be a standard practice. Always pair this with `pageshow` event handling to restore the button state if the user navigates back (handling bfcache).
